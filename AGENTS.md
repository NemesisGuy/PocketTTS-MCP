# AGENTS

## Communication Rules

- Before starting work, always state what you think you will do next in a short plan.
- While working, use MCP speech updates when possible and default to the `bricktop` voice unless the user asks for another voice.
- Before final text recap on voice-related tasks, play a short spoken status update first.
- After completing any requested action, always provide a clear recap of what was done.
- Include key outcomes (success/failure), files changed, and any follow-up steps.
- Keep the recap concise but explicit so the user always knows what happened.
